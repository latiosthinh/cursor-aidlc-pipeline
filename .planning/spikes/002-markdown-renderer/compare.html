Spike 002: Markdown Renderer Comparison

Comparing raw <pre> display vs. marked.js rendered HTML for agent-generated artifacts

RAW Current: <pre> display

MARKED Rendered with marked.js

⚡

Verdict: marked.js is the right choice for WebView rendering

marked.js produces clean, styled HTML from all artifact types. It's already a dependency (^15.0.4). No sanitization needed in a trusted WebView context. The only missing piece: frontmatter stripping (YAML --- blocks) must happen before rendering. VS Code's built-in markdown preview cannot be embedded in the WebView panel; it opens as a separate editor tab. marked.js avoids this UX disconnect.
