# Milestone v1.1: Polish + Post-Hackathon

**Status:** ✅ SHIPPED 2026-05-07
**Phases:** 6-10
**Total Plans:** 5 phases implemented

## Overview

Hardened the engine, unified branding, deepened the reviewer, added skill versioning, made the DAG Kanban-style, and added power features (parallel execution, resume, dry-run).

## Phases

### Phase 6: Engine Hardening
**Goal**: Core engine hardened with graph-based cascade rollback, validated model selection, and safe command execution.
**Depends on**: Phase 5
**Requirements**: CASCADE-01, CASCADE-02, CASCADE-03
**Plans**: 2 plans

Plans:
- [x] 06-01: Graph-based cascade rollback — `findRollbackTarget()` traverses dependency graph
- [x] 06-02: Model validation + command sandbox — real model enum, freeform override, allowlist settings

**Details:**
- CascadeRejector.findRollbackTarget() walks dependency graph to find nearest ancestor
- Default fallback: immediate predecessor when no graph ancestor found
- package.json model enum: cleaned to Claude Sonnet 4, Haiku, GPT-4o, GPT-4o Mini, Gemini 2.0 Flash, Gemini 2.5 Pro
- `aidlc.modelOverride` setting: freeform text override field
- `aidlc.allowedCommands` setting: glob pattern array for command allowlist
- `aidlc.commandConfirmation` setting: user confirmation toggle

### Phase 7: Branding + Deeper Reviewer
**Goal**: Extension unified under AIDLC branding; auto-reviewer supports custom validators and file-existence checks; loop context accumulates across retries.
**Depends on**: Phase 6
**Requirements**: BRAND-01, REVIEW-01, REVIEW-02
**Plans**: 3 plans

Plans:
- [x] 07-01: AIDLC branding — package name, publisher, command prefix, config keys all unified
- [x] 07-02: Deeper auto-reviewer — CustomValidator support, file-existence checks
- [x] 07-03: Loop context accumulation — critic feedback persists across retry iterations

**Details:**
- Package: `aidlc`, publisher: `aidlc`, commands: `aidlc.*`, config: `aidlc.*`
- AutoReviewer supports CustomValidator[] with ValidatorContext
- File-existence checks via extractFileReferences()
- LoopManager accumulates all prior critic feedback into retry agent context

### Phase 8: Skills + Gate UX
**Goal**: Skills support version pinning and selective per-agent injection; gate approval has defined UX behavior.
**Depends on**: Phase 7
**Requirements**: SKILL-01, UX-01
**Plans**: 2 plans

Plans:
- [x] 08-01: Skill versioning + selective injection — version field, targetAgents filter
- [x] 08-02: Gate UX — gate timeout setting, command palette approve/reject

**Details:**
- SkillEntry: `version` (semver), `targetAgents` (string array)
- `buildContextForAgent()` filters skills per agent
- `aidlc.gateTimeout` setting for auto-reject on timeout
- Command palette: `aidlc.approveStep`, `aidlc.rejectStep`

### Phase 9: Interactive DAG
**Goal**: DAG canvas supports interactive exploration with Kanban-style layout and markdown report export.
**Depends on**: Phase 8
**Requirements**: UX-02, UX-03
**Plans**: 2 plans

Plans:
- [x] 09-01: Kanban-style DAG layout — fixed column, snap-to-grid, no free-form dragging
- [x] 09-02: Interactive nodes — hover tooltip, click for artifact, retry count display

**Details:**
- ReactFlow: `nodesDraggable={false}`, `snapToGrid`, `snapGrid={[170, 170]}`
- Nodes in vertical Kanban column with step index display
- Hover tooltip: "Click to view artifact · Retries: N · Loop: mode"

### Phase 10: Power Features
**Goal**: Pipeline supports parallel DAG execution, resume from crash, and dry-run validation without API calls.
**Depends on**: Phase 9
**Requirements**: POWER-01, POWER-02, POWER-03
**Plans**: 3 plans

Plans:
- [x] 10-01: Parallel DAG execution — findParallelGroups() detects independent branches
- [x] 10-02: Resume from crash — resumeRun() loads last incomplete run
- [x] 10-03: Dry-run mode — runDryRun() validates and estimates cost

**Details:**
- PipelineDefinition execution mode: `"sequential"` | `"parallel"`
- `findParallelGroups()` in PipelineValidator: groups independent steps
- `resumeRun()`: loads last failed/paused run, restarts from first incomplete step
- `runDryRun()`: validates config, resolves dependencies, estimates token cost
- Commands: `aidlc.resumeRun`, `aidlc.dryRun`

## Milestone Summary

**Key Decisions:**
- Graph traversal for cascade rollback (not hardcoded N-2)
- Real model enum with freeform override (not validated at runtime)
- Kanban-style DAG over free-form node placement (user preference)
- Sequential parallel opt-in mode (backward compatible)

**Issues Resolved:**
- Cascade rollback precision (graph-traversal)
- Fake model names in settings
- Branding consistency (specflow → AIDLC)
- run_command security hole (allowlist + confirmation)
- Shallow reviewer (file-existence, custom validators)
- Loop context loss (accumulated feedback)
- Skills always injected (selective per-agent)
- Free-form DAG dragging (Kanban layout)

**Issues Deferred:**
- Token/cost budget tracking (v2)
- Run retention/cleanup policy (v2)
